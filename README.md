# CS-300 

What was the problem you were solving in the projects for this course?

The objective of ABCU's CS department was to transition an advising process from a manual system into an automated command -line tool. The program was made to parse an external data file containing course details such as course codes, titles, and prerequisite classes. Then load them into a clean data structure and then handle user queries through a menu. 

How did you approach the problem? Consider why data structures are important to understand.

To determine which data structure was the best for project 2 the algorithmic complexities were evaluated and the three data structures explored in the milestones (Vector, Hash Table, and Binary Search Tree) were compared to see which one was the best fit. After looking at the pros and cons of all three data structures, it was determined that a Binary Search Tree was the ideal choice for the parameters of the project. 

How did you overcome any roadblocks you encountered while going through the activities or project?

One major roadblock I ran into was correctly parsing comma-separated strings into the prerequisites. I overcame this challenge by doing incremental debugging and referencing learning materials to help me better understand my problem. 

How has your work on this project expanded your approach to designing software and developing programs?

These projects helped me better understand writing efficient code. Before I would just write lines of code, but now I understand the importance of evaluating a software’s requirements through Big O. Doing that before writing a single line of code ensures less bottlenecks in performance.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

The main way my programs have improved is to make sure I leave plenty of comments for myself to come back to. I worked on all projects over multiple days and found that I would be lost when coming back to the code. I finally understood that writing in line comments is not just for the professor. It helps me better understand what each section of code is doing, and to stay organized.
